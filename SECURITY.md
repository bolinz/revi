# Security Policy

## Reporting A Vulnerability

Do not open a public GitHub issue for a suspected security vulnerability.

Report vulnerabilities privately through GitHub Security Advisories for this repository or contact the maintainer directly before public disclosure.

Include:

- affected version
- impact summary
- reproduction details
- suggested fix or mitigation if known

## Scope

Security reports are especially relevant for:

- command execution and shell invocation behavior
- git or GitHub bootstrap flows
- generated workflow files that could introduce unsafe defaults
- any behavior that could expose secrets or write to unintended paths

## Disclosure

Validated issues should be fixed privately first and disclosed publicly after a patch or mitigation is available.
